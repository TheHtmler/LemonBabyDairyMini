# 微信小程序官方信息面

只在需要时读取本文件。用于确定应优先查哪些官方页面，以及如何给出有依据的结论。

## 首选来源

1. 开发文档：微信开放文档中的小程序文档、API、组件、框架、云开发、开放能力页面。
2. 更新日志：能力新增、行为变更、废弃说明、基础库要求变化。
3. 运营与审核规则：小程序运营规范、平台运营规则、类目资质、内容规范、广告规则、交易规则。
4. 隐私与个人信息：隐私保护指引、用户信息收集规则、用户授权说明、隐私弹窗相关要求。
5. 官方示例与社区：仅作为补充佐证，不覆盖正式文档和规则页。

## 取证顺序

### API、组件、配置能力

先看：

1. API 或组件正文
2. 参数、返回值、注意事项
3. 最低基础库、平台说明、兼容性说明
4. 相关更新日志

### 审核、合规、权限

先看：

1. 平台规则和审核规范
2. 隐私保护与用户信息规则
3. 行业类目、资质、特殊能力申请条件
4. 与该能力关联的 API 文档注意事项

### “最新是否支持”类问题

先看：

1. 当前官方文档正文
2. 更新日志或最近公告
3. 若文档未写清，再查官方社区是否有官方答复

## 边界原则

- 不把第三方博客、问答、论坛经验当成最终依据。
- 不把开发者工具表现直接等同于真机行为。
- 不把历史版本经验当成当前规则。
- 对未写明的能力边界，宁可保守表述并提示复核，也不要武断承诺。
