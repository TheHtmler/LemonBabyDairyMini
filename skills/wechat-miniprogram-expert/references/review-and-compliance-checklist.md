# 微信小程序审核与合规检查清单

只在涉及审核、上线、功能敏感性、用户信息处理、平台规则时读取本文件。

## 审核前必查

- 功能是否与小程序类目、主体资质、实际服务一致。
- 首页、核心路径、提审账号是否能稳定进入主要功能，不出现空白页、报错或测试环境数据缺失。
- 是否存在外链导流、诱导分享、诱导关注、诱导下载 App、绕过微信交易体系等高风险设计。
- 文案、图片、用户协议、隐私政策、客服入口、联系信息是否完整且一致。
- 是否使用未开放能力、误导性按钮、伪系统弹窗、伪官方标识。

## 权限与隐私

- 每一项用户信息采集都应有明确业务目的、最小必要性说明和前端告知。
- 定位、相册、摄像头、麦克风、蓝牙、通讯录、剪贴板、手机号等敏感能力，要检查授权时机是否与业务动作强关联。
- 若涉及隐私弹窗或隐私协议展示，确认文案、触发时机、拒绝后的降级路径完整。
- 不要在未说明目的前请求高敏权限；避免首次启动即集中索权。

## 常见高风险场景

- 医疗、母婴、儿童、健康、教育、金融、社交、直播、UGC、内容社区、虚拟商品、会员订阅。
- 这些场景除技术实现外，还要额外确认类目、资质、内容审核、支付路径、用户投诉与风控要求。

## 兼容性前必查

- 最低基础库是否覆盖线上目标用户。
- 新 API 是否有旧版本降级路径。
- 是否仅在开发者工具可用、真机存在差异。
- 是否依赖特定客户端版本、系统版本或灰度能力。
- 页面栈、分包、web-view、插件、workers、渲染层行为是否有平台差异。

## 输出建议

给用户结论时，优先输出：

1. 能不能做
2. 哪些条件下能做
3. 审核是否有明显风险
4. 上线前需要补的材料或设计
5. 如果不能直接做，最稳妥替代方案是什么
